# Sayo's Journal のページ構成と、情報を送ってもらうまでの流れ

紗代さんへの説明用。2026-07-11 時点。

---

## 1. いま、ページはこうなっています

Sayo's Journal には「読むためのページ」と「情報を送ってもらうためのページ」があります。
ここでは後者、つまり **窓口のページ** の話をします。

```
トップページ（sayo-kotoba.com）
│
├── 投稿記事の窓口 …………… /request/post
│    └── 投稿の選び方診断 …… /request/post/guide   ← 今回あたらしく作りました
│
├── 取材記事の窓口 …………… /request/interview
│
└── ビジネス向けの窓口 ……… （これから作ります）
```

「窓口ページ」は、ふつうの記事ページとは見た目が違います。
ヘッダーもフッターも専用のものになっていて、**1枚のチラシのように、上から下まで読むと申し込みたくなる**
形に作ってあります。こういうページを、Web の世界では「ランディングページ（LP）」と呼びます。

---

## 2. 「Sayo's Journal を宣伝する親ページ」は作りません

ご相談いただいた「親のLPを1枚作って、その下に4枚ぶら下げる」という案について、
**作らないほうがよい**と判断しました。理由をご説明します。

### すでに親のページがあるからです

**トップページ**と**About ページ**が、まさに「Sayo's Journal とは何か」を伝える場所です。
とくにトップページは、Google からいちばん評価されやすい、サイトの中で最も強いページです。

ここに加えて「Sayo's Journal を宣伝するLP」を別に作ると、
**トップページと、その新しいLPが、同じ言葉で検索されたときに競い合います。**
Google から見ると「どっちを見せればいいの？」という状態になり、
結果としてどちらも中途半端な順位に落ち着いてしまいます。

これは Web では有名な失敗パターンで、「共食い」と呼ばれることもあります。

### 親子関係は、ページを増やさなくても作れます

Google はページの上下関係を、**新しいページではなく、リンクのつながり方**で理解します。

なので、やるべきことはこちらです。

- **トップページに「4つの窓口」への案内を置く**
  → トップページが親として働き、その強さが各窓口に流れていきます。新しいページは1枚も増えません。
- **各窓口ページから、サイト本体（トップ・ブログ・ギャラリー）へ戻るリンクを増やす**
  → 窓口ページが「サイトの一部」だと認識されるほど、サイト全体の評価が窓口にも乗ります。
- **About ページの「3つの記事のかたち」を、窓口が増えたぶん書き足す**

これで、紗代さんがイメージされていた親子構造が、そのまま実現します。
**5枚目のLPに込めたかった内容は、トップページを充実させる形で実装するのが、いちばん効きます。**
作業量は、LPを1枚作るのとほとんど変わりません。

## 3. 投稿記事の窓口（/request/post）を作り直しました

### 変えた理由

このページの読者は、**2種類**います。

1. **SNS に投稿している人**（Instagram、X、Facebook など）
2. **チラシで宣伝している人**（SNS はやっていない）

ところが、これまでのフォームは **SNS の URL が必ず1つ以上必要**でした。
つまり **2番目の人は、そもそも申し込めなかった**のです。ここを直しました。

### いま、どちらか一方でよくなりました

フォームの「送るもの」は、こうなっています。

```
  ┌─────────────────────────────┐
  │  SNS 投稿の URL（最大5件）      │
  └─────────────────────────────┘
              または
  ┌─────────────────────────────┐
  │  チラシ・写真（最大2点）         │
  └─────────────────────────────┘
```

- **どちらか一方**があれば送れます。両方あっても大丈夫です。
- 何も無いときだけ、エラーになります。
- **チラシは PDF でも送れます。** 写真に撮ったものでも構いません。

「または」で区切ってあるので、**文章を読まなくても「どちらか一方でいい」と形で伝わります。**

### SNS の URL は「同じ話題を5つ」です

5つの入力欄が「バラバラの話題を5つ送る枠」だと誤解されないように、こう作りました。

- 各行に **1〜5 の番号**がついています。
- 追加ボタンが「**同じ話題の投稿を追加**」という言葉になっています。
- 「たとえば、飲食店の紹介記事にするなら」という**作例**が開けます。
  （投稿1＝お店の特徴／投稿2＝営業日／投稿3＝昼と夜の使い方／投稿4＝店主の人柄／投稿5＝基本情報）
- 「お店の紹介」と「別のイベント告知」のように**話題が違うものは、分けて送ってください**と書き添えました。

つまり **「バラバラだった5つの投稿を、1本の記事にまとめる」**という価値が、
フォームの見た目から伝わるようにしてあります。

### 写真の使い方を、はっきり書きました

同意チェックの上に、こう書いています。

> 送っていただいた SNS 投稿の文章・写真や、チラシの内容は、記事にする際に使わせていただく場合があります。
> **掲載前に内容をご確認いただき、写真の使用可否もそのときに伺います。**

勝手に使うのではなく、**必ず確認してから使う**という書き方にしました。
よくある質問にも同じ趣旨を追加しています。

## 4. 「投稿の選び方診断」を作りました（/request/post/guide）

### これは何のためのページか

送る人が、いちばん困るのはここです。

> **「5つ送れるのはわかったけど、どの投稿を選べばいいの？」**

その答えを出すページです。3つほどの質問に答えると、
**あなたに合った「5つの投稿の選び方」**がわかります。

大事なのは、**「業種」ではなく「どの魅力を前に出したいか」で分かれる**ことです。
同じ飲食店でも、人柄を伝えたい人と、メニューの多さを見せたい人では、選ぶべき投稿が違うからです。

### 診断で出る「7つの型」

| 型 | どんな記事になるか |
|---|---|
| 1. はじめまして紹介型 | 初めての人に、お店・活動の全体像が伝わる |
| 2. 人柄・ストーリー型 | 「この人に会ってみたい」と思ってもらえる |
| 3. ラインナップ充実型 | 「こんなに選べるんだ」と感じてもらえる |
| 4. こだわり深掘り型 | 「なぜそれが良いのか」の理由まで伝わる |
| 5. 空気感・居心地型 | 「ここなら行けそう」と感じてもらえる |
| 6. 安心案内型 | 料金・駐車場・申込がわかり、不安が減る |
| 7. 参加したくなる型 | 当日のイメージがわき、「行ってみたい」となる |

型が決まると、**その型で送るとよい5つの投稿**が具体的に出ます。

たとえば「人柄・ストーリー型」ならこうです。

1. 自己紹介・プロフィールの投稿
2. 始めたきっかけを書いた投稿
3. 大切にしていることを語った投稿
4. 印象に残っている出来事の投稿
5. これから来る人へのメッセージ

**「新しく文章を書いてください」とは、一言も言っていません。**
すでに投稿してあるものの中から、**どれを選ぶか**の目安です。ここが今回いちばん気をつけたところです。

チラシの方には、「この5つがチラシに載っているかを確かめる目安」として使えます。

### 診断のあと、どうなるか

結果画面には、2つのボタンがあります。

- **「この型で投稿記事を送る」** → 投稿フォームへ移動します。
  このとき、**選んだ型が一緒に運ばれます。**
  フォームの一番上に「診断結果：人柄・ストーリー型」というカードが出て、
  送るべき5つの投稿のリストが表示されます。**答えを持ったまま窓口に座れる**形です。
- **「ほかの型と見くらべる」** → 7つの型の一覧まで移動し、**自分の型のカードが光ります。**
  他の型と見くらべて、選び直すこともできます。

そして、**選ばれた型は紗代さんにも届きます。**
通知メールと、管理画面の依頼詳細・記事化画面に「切り口」として表示されます。
**記事を書き始める前から、その人がどの魅力を前に出したいかがわかります。**

### なぜ、診断ページに投稿フォームを付けなかったか

「診断ページにもフォームを置けば、その場で送れて楽なのでは」と考えられるかもしれませんが、
**付けないほうがよい**と判断しました。理由は3つあります。

1. **同じフォームが2つのURLにできてしまい、検索で共食いします。**
2. **診断が飛ばされます。** 開いた瞬間にフォームが見えると、「診断しなくていいや」となる人が出ます。
   診断ページの存在理由（迷っている人を助ける）が、自分で作った抜け道でなくなってしまいます。
3. **保守が二重になります。** 迷惑メール対策や送信完了画面などを、2箇所ぶん見る必要が出ます。

そのかわり、**行き来はできるようにしてあります。**
投稿フォームのページにも「どの投稿を送ればいいか迷ったら」という案内があり、
そこから診断に寄り道できます。どちらから入っても、行き止まりにはなりません。

## 5. これからのこと

### ビジネス向けLP（4枚目）

企業・団体向けの「Web機能を導入したい」相談窓口です。
これは投稿記事・取材記事と**お客さんも売るものも違う**ので、独立したページとして作る価値があります。

ただし、**新しい申し込みフォームが必要**です。何を聞くか（会社名・担当者・相談内容・予算感など）を
紗代さんと相談してから作ります。

**打ち合わせのときに決めておいてほしいことが1つあります。**
いただいたデモの案には「**カレンダー**」と「**AI検索**」が機能として出てきますが、
- **カレンダー機能は、まだ作っていません。**
- **AI検索は、いまは管理者しか使えません**（一般の方には案内だけが見えます）。

商談でデモをお見せすると、必ず「実際に見せてください」と言われます。
**先に作るのか、デモでは「これからの機能」として分けて見せるのか**を決める必要があります。

### イベントカレンダー

飯田市はイベント情報がまとまった場所がなく、役所も部署ごとにバラバラに出しているので、
市民が「今週末どこで何があるか」を一覧できない ——
このお話をうかがって、**カレンダーは Sayo's Journal の柱になる機能**だと理解しました。
こちらも紗代さんと詰めてから作ります。

---

## まとめ

- **親のLPは作りません。** トップページと About がその役割です。そこにリンクを足すほうが、Google からの評価が強くなります。
- **投稿記事の窓口は、SNS の人にもチラシの人にも開きました。** どちらか一方を送れば申し込めます。
- **診断ページは、「どの投稿を選べばいいか」への答えです。** 結果は投稿フォームまで運ばれ、紗代さんにも届きます。
- **次はビジネス向けです。** フォームの項目と、カレンダー／AI検索の扱いを決めてから作ります。
