# Ticket 21: ルーティング & ナビゲーション再構成

> **優先度**: Phase 2a（最優先）
> **依存**: なし
> **ブロック**: 25

---

## 概要

Phase 2 でサイトのトップページを AI チャットページに変更するため、既存のホームページ（記事一覧）を `/blog` に移動し、ナビゲーション・SEO を更新する。

---

## 実装内容

### 1. ルーティング変更

| パス | 変更前 | 変更後 |
|------|--------|--------|
| `/` | 記事一覧（ホーム） | AI チャットページ（Ticket 25 で実装） |
| `/blog` | なし | 記事一覧（旧ホームを移動） |
| `/search` | 検索 | 変更なし |
| `/[category]/[slug]` | 記事詳細 | 変更なし |

### 2. ファイル移動

**記事一覧を `/blog` に移動**:

```
src/app/(public)/page.tsx          → src/app/(public)/blog/page.tsx
```

- `page.tsx` の内部ロジック・コンポーネントはそのまま
- メタデータを更新（title: "ブログ | Sayo's Journal" 等）

**AI チャットページのプレースホルダー作成**:

```
src/app/(public)/page.tsx          → 新規作成（スケルトン）
```

- Ticket 25 で本格実装するため、最低限の構造のみ
- 「AI チャット準備中」的な表示 + `/blog` へのリンク

### 3. ナビゲーション更新

**Header (`src/components/Header.tsx`)** のナビゲーションリンク:

| 変更前 | 変更後 |
|--------|--------|
| ホーム (`/`) | ホーム (`/`) |
| - | ブログ (`/blog`) |
| About (`/about`) | About (`/about`) — 未実装のまま |

- モバイルメニューも同様に更新

### 4. 内部リンクの更新

以下のコンポーネントで「ホームに戻る」系のリンクを確認・更新:

- Footer のリンク
- 検索ページの「ホームに戻る」リンク
- FilterBar のリセット時の遷移先
- パンくずリストの「ホーム」リンク
- 利用制限到達時のフォールバックリンク（→ `/blog`）

> **注意**: 記事一覧への誘導は `/blog` に、サイトホームへの誘導は `/` に。使い分けを明確にする。

### 5. SEO 更新

**サイトマップ (`src/app/sitemap.ts`)**:
- `/` を追加（AI チャットページ、priority: 1.0）
- `/blog` を追加（記事一覧、priority: 0.8, changeFrequency: daily）
- 既存の記事 URL は変更なし

**メタデータ**:
- `/`: AI チャットページ用のメタデータ
- `/blog`: 記事一覧用のメタデータ（旧ホームの内容を引き継ぐ）

**robots.txt**: 変更なし（AI チャットの動的会話内容はクロール対象外 — JS レンダリングされないため自然に対象外）

### 6. ISR / キャッシュ設定

| ページ | 設定 |
|--------|------|
| `/` (AI チャット) | Dynamic（キャッシュなし） |
| `/blog` (記事一覧) | ISR revalidate = 600（旧ホームと同じ） |

- `src/app/(public)/blog/page.tsx` — 新規（旧 page.tsx の移動先）

- `src/components/Footer.tsx` — リンク確認
- `src/app/sitemap.ts` — エントリ追加
- FilterBar, Breadcrumbs, 検索ページ — 内部リンク確認
